## 用户登录与注册

这里假定用户名是唯一的

注册register的过程: 

生成一个随机的加密盐, 将 用户名, 加密后的密码, 随机盐 存入数据库

登录login的过程: 

查找用户名, 获取用户加密盐, 通过用户输入的密码+盐, 得到加密后的密码

用这个密码查询数据库, 如果找到对应的记录, 则说明密码正确

登录完成后, 需要保留一个 session 状态, 表示用户已经登录过

可以将用户名存储在 session 中, 便于其他接口使用这个数据查找用户信息, 以此关联相关数据

登出logout: 

清除session即可

这里把需要的知识拼接起来, 就可以实现我们想要的功能了

关于 session 的使用, 参考 [session_validator](session_validator.md '')

## 密码加密

安装加密库 

```
npm install crypto-js
```

核心原则是 数据库里面不要存明文

那么如何选择加密算法, 使用什么加密算法, 是个比较复杂的问题

这里给出最简单的答案, 使用 pbkdf2 加密算法

这个加密方法需要额外提供两个东西

- 加密盐, 可以简单理解有盐可以更安全, 我们把盐也存到数据库中

- 迭代次数, 可以简单理解成增加计算量, 计算量越大, 破解成本越高

```JavaScript
import {pbkdf2, randomBytes} from "crypto"

const salt = randomBytes(16).toString('hex')


// 迭代次数  越多越慢
const iterations = 50000
// 生成的密钥字节数
const keylen = 64
// 加密算法
const digest = "sha512"
// pbkdf2是个异步函数，并且哈希值在callback中返回的
const result = new Promise<string>((resolve, reject) => {
    pbkdf2(password, salt, iterations, keylen, digest,
    (err, derivedKey) => {
        if (err) {
            reject(err)
        } else {
            // 返回最终哈希值
            const hash = derivedKey.toString('hex')
            resolve(hash)
        }
    })
})


```
