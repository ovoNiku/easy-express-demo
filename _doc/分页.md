# 分页

从后端角度看, 分页的核心是, 后端通过 limit 返回指定的分页的数据

比如传页码 = 0, 那就选择 0~10 的数据

传递页码 = 1, 那就选择, 11~20 的数据

...

从前端角度看, 分页器的核心是: 

展示内容, 展示页码, 找出当前页码, 点击页码可以更新

# 动态获取数据

JS 发送 ajax 请求的方式, 更新了很多代

从最早的 XMLHTTPRequest 到现在的 fetch

这里我们只需要一个 get 请求服务端的功能, 参考代码如下

```JavaScript
fetch("http://example.com/movies.json")
  .then((response) => response.json())
  .then((data) => console.log(data))
```

# 什么是委托

js dom 的点击有一个冒泡机制, 比如下面的结构

```html
<div class='top'>
    <div class='middle'>
        <div class='small'>
            点击1
        </div>  
        <div class='small'>
            点击2
        </div>  
    </div>
</div>
```

如果点击了 small, middle 和 top 也会被触发点击

基于这个机制, 我们可以把事件绑定在 middle 上面, 判断实际触发点击的对象然后执行对应的方法

这样的好处有两个 

- 只需要给 middle 绑定一次, 不需要每一个 small 都绑定一次
- 如果子节点是动态生成的, 可以在生成之前就做好事件的绑定

这种绑定事件的技巧叫做委托
